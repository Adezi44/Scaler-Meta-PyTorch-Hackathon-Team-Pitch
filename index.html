01 — The core idea

We're not discovering drugs.
We're building the gym.

Drug discovery is the process of finding a molecule that fits a disease-causing protein — like finding a key for a specific lock. There are 10⁶⁰ possible drug-like molecules — more than atoms in the universe — so you can't brute-force it. You need an AI agent that learns to navigate that space intelligently.

Our environment simulates exactly that search loop. An agent proposes a molecule, our env scores it (does it bind? is it safe? can it be synthesized?), and the agent refines. This maps perfectly onto the OpenEnv step() / reset() / state() loop.

The clearest analogy

We're not building a real aircraft. We're building a flight simulator — realistic controls, realistic physics, real feedback. Pilots learn to fly in simulators. Agents will learn drug discovery in ours.

03 — The three tasks

Easy → medium → hard.
All objectively gradable.

The OpenEnv spec requires minimum 3 tasks with graders scoring 0.0–1.0. Here's our task ladder — each has a concrete, automatic grader, no manual review needed.

Easy

Lead optimization

Agent gets a near-perfect molecule. Tweak it to push binding affinity from ~200 nM to below 50 nM without breaking drug-likeness. Grader: affinity score + QED score.

Medium

Scaffold hopping

Agent gets a potent molecule with a toxicity flag. Find a structurally different molecule that retains potency but clears toxicity. Grader: toxicity cleared + affinity within 5×.

Hard

De novo design

Only the protein pocket is given. Agent designs from scratch in 15 iterations, balancing potency, selectivity, solubility, and synthesizability. Grader: composite score across all four axes.

All graders run automatically using open-source tools. Scores are fully deterministic — no ambiguity.

04 — What we actually build

Three pip installs.
No wet lab required.

The entire environment runs on open-source tools within the 2 vCPU / 8GB RAM constraint. The chemistry complexity is hidden inside libraries — we just call functions and get scores back.

rdkit
Validates molecules, computes QED (drug-likeness) and SA (synthesizability) scores. Industry standard, CPU-only.
GNN binding proxy
Lightweight pretrained graph neural net from ChEMBL data. Approximates binding affinity in milliseconds.
ChEMBL dataset
2M+ molecules with real experimental data. Our proxy scores are grounded in actual science.
pydantic + FastAPI
Typed models and REST endpoints for OpenEnv spec compliance: step(), reset(), state().

06 — Handling the hard questions

Every objection,
answered.

How is this different from what DeepMind does?
DeepMind is trying to discover actual drugs. We're building the training environment where agents learn medicinal chemistry reasoning. AlphaFold solved protein structure — the molecule design step is still open. That's our space.
Do we need a biology or chemistry background to build this?
No. The chemistry lives inside RDKit — we call .score(smiles) and get a number. The environment is Python classes, FastAPI endpoints, and JSON. We're engineers building a simulation, not scientists running experiments.
Are the grader scores scientifically valid?
Yes. QED was published in Nature Chemistry. SA score came from Novartis researchers. The GNN proxy is trained on real ChEMBL experimental data. These aren't toy metrics — they're used in actual drug programs.
Will it run under 20 minutes on 2 vCPU / 8GB RAM?
Easily. RDKit is CPU-only and extremely fast — each molecule evaluation takes under 100ms. Even the hard task with 15 iterations completes in under a minute. The inference script will finish in minutes, not hours.
